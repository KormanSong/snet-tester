# UI Verification & Validation Methodology

## Problem Statement
XML/코드 구조 분석만으로는 실제 화면의 잘림, 겹침, 가독성 문제를 발견할 수 없다.
레이아웃 계산이 이론적으로 맞더라도, 런타임 렌더링에서 폰트 메트릭, DPI 스케일링,
위젯 내부 패딩, pyqtgraph 축 레이아웃 등의 차이로 실제 화면이 다르게 나타난다.

## 잘못된 검증 방법 (Anti-Pattern)
- .ui XML만 읽고 "크기가 맞으니 통과"라고 판단
- 픽셀 계산을 종이 위에서만 수행
- 실제 앱을 실행하지 않고 "이론적으로 문제없음" 결론

## 올바른 검증 단계

### Phase 1: 스크린샷 기반 검증
1. `uv run main.py` 실행
2. 실제 창의 스크린샷을 캡처 (또는 `--export` 옵션 사용)
3. 모든 텍스트/라벨이 잘리지 않고 완전히 보이는지 육안 확인
4. 그래프의 축 라벨, 눈금 숫자, 범례가 완전히 표시되는지 확인

### Phase 2: 경계 조건 검증
1. 창을 최소 크기(1120x760)로 줄여서 확인
2. 창을 최대화하여 확인
3. 각 상태에서 텍스트 잘림, 위젯 겹침, 스크롤바 발생 여부 확인

### Phase 3: 동적 상태 검증
1. 데이터가 흐르는 상태에서 그래프 확인
2. 최대값(100%), 최소값(0%), 중간값이 축 눈금에서 모두 보이는지
3. 채널 6개 모두 활성화 시 범례/색상 구분 가능한지

### Phase 4: Designer 일치 검증
1. `uv run snet-designer` 실행
2. 런타임 앱과 나란히 비교
3. 위젯 비율, 폰트 크기, 색상이 일치하는지 확인

### Phase 5: pyqtgraph 특수 검증
pyqtgraph 위젯은 .ui로 제어할 수 없으므로 특별히 주의:
- 축 라벨(setLabel): 텍스트가 잘리지 않는지
- 축 너비(setWidth): 숫자 + 라벨이 충분히 들어가는지
- tickTextWidth: 가장 긴 숫자(예: "100")가 잘리지 않는지
- setContentsMargins: 좌측/우측 여백이 축 라벨을 가리지 않는지
- showValues: 시간축 숫자가 표시되는지
- 그리드 alpha: 실제로 눈에 보이는 수준인지

## Checklist (신규 그래프 변경 시 필수)
- [ ] RATIO% 라벨 완전히 보임
- [ ] VALVE 라벨 완전히 보임
- [ ] Y축 최대값(100) 숫자 보임
- [ ] Y축 최소값(0) 숫자 보임
- [ ] X축 시간 숫자 보임 (0, 2, 4, 6, 8, 10)
- [ ] 그리드 라인 육안으로 확인 가능
- [ ] 6개 채널 색상 구분 가능
- [ ] TX(설정값, 점선) vs RX(실측값, 실선) 구분 가능
- [ ] 창 최소 크기에서 잘림 없음
- [ ] Designer와 런타임 동일
